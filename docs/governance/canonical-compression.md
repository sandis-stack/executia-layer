# Canonical Compression (Phase 6A)

Quieter, stronger institutional presentation across EXECUTIA — compression and stabilization only. No API, SQL, or execution logic changes.

## Objective

As governance layers mature, presentation compresses:

- simpler vocabulary
- calmer bands
- stronger inevitability
- preserved sovereignty, trust, intent, memory, consequence, rhythm

Not concept-heavy, descriptor-heavy, or philosophically verbose.

## Compression layer

| Asset | Role |
|-------|------|
| `executia-canonical-compression.js` | Canonical language core, `applyCompression`, unified posture block |
| `executia-canonical-compression.css` | Quiet band spacing, hide redundant framing |

Enable with `ex-compression-enabled` (engineering console default).

## Canonical language core

| Token | Use |
|-------|-----|
| Canonical execution | Primary posture |
| Governed · calm · inevitable | Inevitability line |
| META.* | One-line band metadata per layer |

Display caps: frame rows hidden; state/frame lists limited to 3.

## Engineering console

```javascript
COMPRESSION().applyCompression(ctx);
```

Re-applies all presentation layers in compressed mode and collapses duplicate surface framing into `canonicalPostureBlock`.

## Preserved (compressed presentation)

Identity, rhythm, consequence, memory, intent, trust, sovereignty — same semantics, shorter surfaces.
