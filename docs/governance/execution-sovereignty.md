# Execution Sovereignty (Phase 5M)

Institutional execution jurisdiction across EXECUTIA surfaces — presentation only. No API, SQL, or execution logic changes.

## Objective

The system must feel:

- sovereign
- system-independent
- canonically authoritative
- execution-jurisdiction aware
- institutionally supreme

Not a governance application, enterprise platform, orchestration software, or operational tooling.

## Sovereignty layer

| Asset | Role |
|-------|------|
| `executia-execution-sovereignty.js` | Jurisdiction framing, sovereignty band, `applySovereignty` |
| `executia-execution-sovereignty.css` | Institutional sovereignty band + jurisdiction blocks |

Imported via `executia-governed-presentation.css`. Enable with `ex-sovereignty-enabled`.

## Engineering console

Sovereignty band mounts below trust; state derived from console authority, canonical anchors, and source presence.

```javascript
SOVEREIGNTY().applySovereignty(ctx);
```

Surfaces use jurisdiction framing:

```javascript
SOVEREIGNTY().jurisdictionFramingHtml(ctx);
SOVEREIGNTY().jurisdictionSurfaceTitle();
SOVEREIGNTY().authoritySurfaceTitle();
```

## Descriptors

- Execution authority sits above infrastructure — governance is sovereign, not subordinate to systems.
- Canonical jurisdiction is permanent; execution truth survives system change.
- Deterministic jurisdiction binds verify, replay, and consequence without platform dependency.
- Institutional execution jurisdiction maintains supremacy across governed transitions.
